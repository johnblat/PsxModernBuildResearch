domes: 実装済みのほとんど全てのポリゴン/共有ポリゴン

＜サンプルの解説＞
	現在実装されている HMD プリミティブのうち、ポリゴン/共有ポリゴンに
	ついて、良く使われるタイプほとんど全てを網羅したデータです。
	(ドライバビット (DRV) については、CLP, ADV, DIV、
	タイプ (PRIM_TYPE) については TILE, PST, MIP, COL, MESH を
	用いたデータは含んでいません)

	共有ポリゴンをわかりやすく示すために、アニメーションが付加して
	あります。HMD サンプルプログラムディレクトリの anim/animview を用いて
	再生してください。

	データは、3x3x4 の三次元マトリックス状に配置された半球群で
	構成されています。

	正面から見た状態で、

		a11:BOT|LGT	a12:LGT		a13:STP|LGT

		a21:BOT		a22:Normal	a23:STP

		a31:BOT|STP	a32:FOG		a33:BOT|STP|LGT

	となっています。

	また半球群は、Z 軸奥へ向かって、Flat, Gouraud, Flat Textured,
	Gouraud Textured の順番に並んでいます。

	各半球は、三角と四角の両方で構成され、中央部分は共有ポリゴンで
	作成されています。

	a21 と a31 に配置された半球では、中央部分が欠けて表示されます。
	これらのデータは、光源計算有りの両面共有ポリゴンです。
	現在の HMD フォーマットの枠組みでは、このタイプで裏面の輝度を
	計算することが困難であるため、データフォーマットとしては定義可能
	ですが、ライブラリではサポートしていません。
	そのため、animview 起動時、コンソールにエラーメッセージが
	出力されますが、真のエラーではありませんので無視してください。

	このデータは、以下の手順で作成されました。

	1. LightWave 3D を用いて、アニメーション付きのデータが作成されました。
	   (このデータは、lw3d ディレクトリに保存されています)
	2. 株式会社ディ・ストーム様により、開発/提供中の "Animation Saver for
	   PlayStation" version 1.1.2 により、LAB ファイルが生成されました。
	   (lw3d/domes.set に、用いた設定が保存されています)
	3. このバージョンの Animation Saver においては、光源計算無し、
	   テクスチャ付き三角形が正しく出力されません。
	   これは、同プラグイン開発時において PlayStation ライブラリが、
	   このタイプをサポートしていなかったためです。
	   そこで、2 で得られた LAB ファイルを編集します。
	4. TRI|LMD|TME タイプのポリゴンデータは、Poly_0004, Poly_0006,
	   Poly_0020, Poly_0022, Poly_0036, Poly_0038, Poly_0132, Poly_0134
	   の各ラベルで示される部分に格納されています。
	   これらのデータパケットの先頭に、R, G, B, CODE を付加しました。
	   (domes.lab では、"added" というコメントで、この部分を識別できます)
	5. 三次元マトリックスの、要素 a32 に FOG 効果を付け足します。
	   PolyPrim_0112 から PolyPrim_0127 と、SharedPolyPrim_0079 から
	   SharedPolyPrim_0127 のドライバビットに、FOG フラグを立てました。
	6. 以上の結果から、HMD アセンブラ (labp) により HMD ファイル
	   "domes.hmd" が生成されました。
