＜サンプルの解説＞

	  このプログラムは、コントローラライブラリ(libpad)を使うサンプル
	である。具体的には２種類のアクチュエータ（振動子）を動作させたり、
	アプリケーションの中からの端末種別切替え、端末種別切替えスイッチの
	ロック／ロック解除を行うことが出来る。またコントローラのアクチュ
	エータ情報の問い合わせ機能を使ってアクチュエータのプリミティブを
	調べ画面上に表示する。

	  またライブラリに追加されたコントローラとの通信中断／再開を
	PlayStation 本体のポートごとに設定する機能も使っている。


	・画面表示の説明
	[MULTI TAP]
		マルチタップ経由でコントローラを接続している時は[MULTI TAP]
		コントローラをPlayStation本体のコントローラ端子に直接接続し
		ている時は[DIRECT]と表示される。

	1A [EX]=7
		PlayStationポート1の接続されているマルチタップのポートAには
		コントローラ(Dual Shock)でコントローラモードID7が接続されて
		いる。
		この他には、次の２種類がある。
		コントローラが接続されていない時:[NONE]、
		コントローラ(Dual Shock)以外のコントローラが接続されている
		とき:[CTRL]

	G=0300,156
		端末種別=3 のガンコントローラが接続されている時のみ表示される。
		ガンの向いている座標を示す。

	B=FFFF
		ボタンの押下状態（各ビットについて 1：リリース、0：プッシュ）

	ST=80,80,80,80
		アナログチャネル値（4チャネル）
		#define MULTITAP 0 の時のみ表示

	ID 4,7
		サポートしているコントローラモードのIDは 4 と 7 の２種類

	AC(1,2,0,10)=0 (1,1,1,20)=90
		・アクチュエータ0は 連続回転型振動(1)、高速回転(2)、
		  データ長1ビット(0)、消費電流100mA(10)、現在の値0
		・アクチュエータ1は 連続回転型振動(1)、低速回転(1)、
		  データ長1バイト(1)、消費電流200mA(20)、現在の値90

	CM 0,1
		同時に制御できるアクチュエータの組み合わせは１通りでその内容は
		アクチュエータ番号0番と1番である。すなわちすべてのアクチュエー
		タが同時に制御できる

	SW ULK
		端末種別切り替えスイッチは使用可能状態（ロックされていない）
		この他には LK:使用禁止状態（ロックされている）がある。

＜起動方法＞ 
	>psymake	コンパイル
	>run main	実行

＜コントローラの操作方法＞
	L1 :アクチュエータ０の値−１
	L2 :アクチュエータ０の値＋１
	R1 :アクチュエータ１の値−１０
	R2 :アクチュエータ１の値＋１０
	○ :コントローラモード切替スイッチ ロック／アンロック
	← :コントローラモードを「１６ボタンコントローラ」に切り替える
	→ :コントローラモードを「アナログコントローラ」に切り替える

＜備考＞
	FntPrint() で表示できる文字数の関係でマルチタップを使ってコントロー
	ラを５個以上接続すると画面上の表示が欠けることがあるが、コントロー
	ラとの通信は正常に行われている。
